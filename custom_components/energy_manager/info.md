# Energy Manager

ניהול אנרגיה סולארית בבית: מצב סוללה, ייצור PV, צריכת בית, תחזית **Open-Meteo + pvlib**, אסטרטגיית סוללה, ו**הפעלה אוטומטית של צרכנים** (מתגים / `input_boolean`) עם השהיות ועדיפויות.

**גרסה:** ראו `manifest.json` באותה תיקייה (שדה `version`). תלויות Python לזמן ריצה מופיעות שם תחת `requirements`.

## תכונות עיקריות

- **חיישנים** — SOC והספק סוללה, ייצור סולארי, צריכת בית, תחזית (שעה הבאה / מחר / נותר), מרווח יומי, שעות לזריחה/שקיעה, מצבי אבחון טעינה/פריקה, טקסט **התראת אינטגרציה אחרונה**, ועוד.
- **תחזית סולארית** — נתוני קרינה משעתיים מ-Open-Meteo, המרה ל-POA והספק עם pvlib, תמיכה ב**מספר מחרוזות PV** (שיפוע, אזימוט, הספק), ומגבול אופציונלי על יציאת הממיר. שמירת סדרת kW שעתית ב-`.storage` של Home Assistant; אם ה-API לא זמין — **בנייה מחדש מהמטמון** כשהסדרה עדיין מכסה את "עכשיו" והגאומטריה תואמת.
- **מנוע החלטות** — מצבים: `saving`, `normal`, `wasting`, `emergency_saving`; רמות אסטרטגיה לסוללה: `low` / `medium` / `high` / `full`. **בלי תחזית** — התאמה לפי **פסי SOC** (ירידה של רמה אחת בכל פעם); ב-**low / very low** נשאר שמרני (`FULL`).
- **גשר לילה** — בערך שעתיים לפני הזריחה, בחושך ובתנאים בטוחים (כולל מול תחזית מחר מול צורך בטעינה), המערכת יכולה לרכך את הכלל "אין PV בשעה הבאה → אסטרטגיה מלאה" כדי שלא להיתקע שמרני לפני הבוקר למרות עודף יומי ארוך.
- **מנהל עומסים (צרכנים)** — הפעלה ב-**wasting** עם השהיה מוגדרת; כיבוי הדרגתי בחזרה ל-**normal**; מכשירי on/off אופציונליים לחיסכון קיצוני; חריגה ממגבלת פריקה יכולה לכבות צרכן אחד.
- **ישויות Select** — כש-**Manual Mode Override** או **Manual Strategy Override** פעיל, בוחרים ערך ב-**Select** המתאים (מצב / אסטרטגיה).
- **Telegram (אופציונלי)** — בתפריט ההגדרות **Telegram**: העברת אירועים בסגנון לוג לצ'אטים, ואופציונלית פקודות דרך בוט (טוקן ו-chat IDs בהגדרות).
- **אבחון ולוג** — לוג טקסט אופציונלי תחת תיקיית הקונפיג של HA (`energy_manager_logs/`); **התראות אינטגרציה** בזיכרון וחיישן **התראה אחרונה**.

## שירותים

ב-**Developer tools → Services** (תיאורי השדות ב-`services.yaml`):

| שירות | תפקיד |
|--------|--------|
| `energy_manager.reset_consumer_learn` | איפוס למידת הספק לפי צרכן; אופציונלי `config_entry_id` לכניסת הגדרות אחת. |
| `energy_manager.clear_integration_alerts` | ניקוי מחסנית ההתראות בזיכרון / חיישן ההתראה האחרונה; אופציונלי `config_entry_id`. |

## דרישות

- Home Assistant (Supervised, Container או OS).
- חבילות Python (בדרך כלל מתקינות HA אוטומטית מ-`manifest.json`): `pvlib`, `aiohttp`, `pandas`.

## התקנה (HACS)

1. **HACS** → **Integrations** → **⋮** → **Custom repositories**.
2. הוסף: `https://github.com/refael302/energy_management`, קטגוריה **Integration**.
3. התקן **Energy Manager** והפעל מחדש את Home Assistant.
4. **הגדרות** → **מכשירים ושירותים** → **הוסף אינטגרציה** → **Energy Manager**.

## הגדרות

הכל דרך ממשק המשתמש (config flow): חיישני סוללה והספק, גאומטריית מחרוזות PV, צריכת בסיס, גורמי בטיחות לתחזית, רזרבת חירום, יעד סוללה לסוף יום, צרכנים, השהיות, עקיפות ידניות, **Telegram**, וכו'.
